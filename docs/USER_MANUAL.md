## 1. 系统简介

### 1.1 系统定位

CageLedger 是一个面向实验动物中心内网使用的轻量级笼位管理与饲养费核算系统，主要解决以下问题：

| 问题 | 系统能力 |
| --- | --- |
| 饲养间、笼架、笼位信息分散 | 按饲养间、笼架、行列统一建立结构化笼位图 |
| 笼位状态依赖人工台账 | 支持空、预约、在用、取材结束等状态维护 |
| 项目和 IACUC 信息重复录入 | 上传 IACUC 汇总表后自动回填项目名称和负责人 |
| 饲养费核算手工计算成本高 | 支持按笼位图自动核算和数量统计表录入核算 |
| 房间管理员权限边界不清 | 房间管理员只看见并只能维护授权房间 |
| 操作缺少追溯 | 后端记录关键写操作的审计日志 |

### 1.2 核心业务对象

| 对象 | 含义 | 示例 |
| --- | --- | --- |
| 饲养间 | 实际房间或动物饲养区域 | `8107`、`SPF 小鼠饲养间 A` |
| 笼架 | 饲养间中的某个架体 | `01 号笼架`、`A 架` |
| 笼位 | 笼架上的具体位置 | `8107-01-A1` |
| 占用记录 | 某个笼位在某段时间内的项目占用信息 | IACUC、负责人、入住日期 |
| IACUC 索引 | 动物实验申请汇总表导入后的项目索引 | 伦理编号、项目名称、负责人 |
| 数量统计表 | 纸质数量统计表电子化录入结果 | 月初结余、每日新增/减少 |
| 结算单 | 按项目负责人和月份生成的计费结果 | CSV 明细、PDF 结算单 |

### 1.3 当前版本能力范围

`v0.3.4` 已支持：

- 共享模式 SQLite 存储。
- 系统管理员与房间管理员账号。
- 房间管理员授权过滤。
- 饲养间和笼架分开维护。
- 非连续笼架编号。
- 笼架规格编辑。
- 动态笼位图预览与弹窗编辑。
- 单笼录入、批量录入、设为空、取材结束。
- IACUC CSV 上传和自动回填。
- 项目负责人身份管理与免费笼数。
- 动态笼位图自动核算。
- 数量统计表录入核算。
- CSV 饲养明细和 PDF 结算单导出。
- GitHub Release、离线包和 GHCR 容器镜像自动发布。

## 2. 快速上手

### 2.1 访问系统

部署完成后，在浏览器访问：

```text
http://服务器IP:5173
```

本机开发或测试时访问：

```text
http://localhost:5173
```

### 2.2 首次登录

默认管理员账号：

| 字段 | 默认值 |
| --- | --- |
| 用户名 | `admin` |
| 密码 | `admin123` |

> [!WARNING]
> 正式使用前应通过环境变量修改默认管理员密码，或登录后创建新管理员账号并停用默认口令。

### 2.3 管理员首次配置建议

| 顺序 | 操作 | 菜单 |

房间管理员登录后通常只需要关注：

| 工作 | 菜单 |
| --- | --- |
| 查看笼位状态 | 笼位图 |
| 录入入住、预约、取材 | 笼位图 |
| 新增或调整授权房间内笼架 | 房间管理 |
| 录入纸质数量统计表 | 饲养费核算 |

## 3. 角色与权限

### 3.1 角色速查

| 功能 | 系统管理员 | 房间管理员 |
| --- | --- | --- |
| 查看全部房间 | 可以 | 不可以，只能查看授权房间 |
| 新增饲养间 | 可以 | 不可以 |
| 删除饲养间 | 可以 | 不可以 |
| 新增授权房间内笼架 | 可以 | 可以 |
| 编辑授权房间内笼架 | 可以 | 可以 |
| 删除授权房间内笼架 | 可以 | 可以 |
| 编辑授权房间内笼位信息 | 可以 | 可以 |
| 编辑未授权房间笼位信息 | 可以 | 不可以 |
| 上传 IACUC 索引 | 可以 | 不可以 |
| 账号管理 | 可以 | 不可以 |
| 系统管理 | 可以 | 不可以 |
| 查看操作日志 | 可以 | 视系统配置显示，但只能看到接口返回内容 |
| 导出正式结算文件 | 可以 | 受当前共享模式权限限制 |

### 3.2 权限过滤逻辑

房间管理员的权限控制同时发生在前端和后端。

| 层级 | 控制内容 |
| --- | --- |
| 前端 | 菜单、房间下拉框、房间卡片、笼架列表只展示授权房间 |
| 后端 | `/api/state`、实体 API、设施批量写入接口按授权房间过滤和校验 |

如果房间管理员尝试通过异常请求修改未授权房间，后端会拒绝写入。

### 3.3 授权原则

建议按实际管理边界授权：

- 一个房间可授权给多个房间管理员。
- 一个房间管理员可管理多个房间。
- 跨区域、跨楼层、跨 SPF 等级的授权应谨慎。

## 4. 页面与菜单说明

### 4.1 首页

首页用于快速查看系统状态，通常包含：

- 总笼位数。
- 在用、预约、空笼数量。
- 房间容量概览。
- 当前系统基本运行信息。

### 4.2 笼位图

用于查看和维护笼位占用。当前版本默认进入预览状态，编辑操作通过弹窗完成。

### 4.3 房间管理

用于维护饲养间、笼架和笼位结构。当前版本默认进入房间结构预览状态，新增饲养间和新增笼架通过弹窗完成。

### 4.4 饲养费核算

包含两个核算入口：

| 页签 | 用途 |
| --- | --- |
| 动态笼位图（自动） | 从笼位占用记录自动计算每日笼数和费用 |
| 数量统计表（录入） | 录入纸质统计表后按每日结余生成计费明细 |

### 4.5 数据管理

用于上传 IACUC 汇总表、维护项目索引和数据相关内容。

### 4.6 账号管理

仅系统管理员可用，用于创建账号、修改角色、授权房间。

### 4.7 系统管理

用于查看系统信息、版本记录、更新检查、API 概览和维护文档入口。

### 4.8 操作日志

用于追溯关键业务操作，包括账号、动作、对象、时间和变更快照。

## 5. 房间与笼架管理

> 入口：`房间管理`

### 5.1 页面布局

房间管理页面默认展示房间和笼架结构，不直接展开编辑表单。

| 区域 | 说明 |
| --- | --- |
| 房间树 | 展示房间、笼架、笼位数量、在用数、预约数 |
| 编辑操作 | 提供“新增饲养间”和“新增笼架”按钮 |
| 弹窗编辑 | 点击按钮后弹出新增表单 |

### 5.2 新增饲养间

适用角色：系统管理员。

操作步骤：

1. 进入 `房间管理`。
2. 点击 `新增饲养间`。
3. 填写饲养间名称。
4. 选填区域，例如 `屏障区`。
5. 点击提交。

字段说明：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 饲养间名称 | 是 | 建议使用实际房间号或规范名称 |
| 区域 | 否 | 可填 SPF、屏障区、普通区等 |

> [!NOTE]
> 新增饲养间不会自动创建笼架。应先建房间，再按实际摆放逐个新增笼架。

### 5.3 新增笼架

适用角色：系统管理员、授权房间管理员。

操作步骤：

1. 进入 `房间管理`。
2. 点击 `新增笼架`。
3. 选择所属饲养间。
4. 输入笼架编号。
5. 输入行数和列数。
6. 提交后系统自动生成笼位。

字段说明：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 所属饲养间 | 是 | 房间管理员只能选择授权房间 |
| 笼架编号 | 是 | 支持非连续编号，如 1、3、7 |
| 行数 X | 是 | 笼架纵向行数 |
| 列数 Y | 是 | 笼架横向列数 |

生成规则：

```text
笼位数 = 行数 × 列数
```

示例：

| 行数 | 列数 | 生成笼位数 |
| --- | --- | --- |
| 10 | 7 | 70 |
| 5 | 6 | 30 |
| 4 | 5 | 20 |

### 5.4 编辑笼架

可编辑内容：

- 笼架编号。
- 行数。
- 列数。

注意事项：

| 情况 | 系统行为 |
| --- | --- |
| 扩大行列 | 新增缺失笼位 |
| 缩小行列 | 如果要删除的笼位仍在用或预约，系统会阻止 |
| 修改编号 | 笼位展示编码会随编号变化 |

### 5.5 删除笼架

操作步骤：

1. 在房间树中找到对应笼架。
2. 点击删除按钮。
3. 阅读确认提示。
4. 确认删除。

删除影响：

- 删除该笼架。
- 删除该笼架下的笼位结构。
- 历史占用记录作为流水保留，不随结构级联删除。

### 5.6 删除饲养间

适用角色：系统管理员。

删除影响：

- 删除饲养间。
- 删除下属笼架。
- 删除下属笼位结构。
- 历史占用记录保留。

> [!CAUTION]
> 删除饲养间属于高风险操作。正式使用环境建议先确认该房间是否已停用，并备份数据库。

## 6. 笼位图与占用录入

> 入口：`笼位图`

### 6.1 状态说明

| 状态 | 含义 | 是否计入在养 |
| --- | --- | --- |
| 空 | 当前没有占用 | 否 |
| 已预约 | 已安排但未正式在养 | 默认不计费 |
| 在用 | 当前正在饲养 | 是 |
| 已结束 | 历史占用记录结束 | 否 |

### 6.2 预览模式

默认进入笼位图预览模式，适合日常查看。

可查看内容：

- 当前房间和笼架。
- 每个笼位状态。
- IACUC、负责人、开始日期等摘要。
- 状态筛选。

### 6.3 打开编辑面板

点击 `编辑笼位图` 打开右侧或居中的编辑弹窗。

编辑弹窗可执行：

- 单笼录入。
- 批量录入。
- 设为空。
- 已取材。
- 查看占用历史。

### 6.4 单笼录入

操作步骤：

1. 点击需要编辑的笼位。
2. 点击 `编辑笼位图`。
3. 填写状态、IACUC 编号、开始日期。
4. 根据需要填写笼盒编号、项目名称、项目负责人、实验负责人、备注。
5. 点击 `保存笼位`。

字段说明：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 状态 | 是 | 空、预约、在用 |
| IACUC 编号 | 在预约/在用时应填写 | 可通过索引自动回填 |
| 项目名称 | 否 | 可由 IACUC 自动带出 |
| 项目负责人 | 否 | 计费用核心字段 |
| 实验负责人 | 否 | 业务联系字段 |
| 开始日期 | 是 | 计费开始依据 |
| 结束/最后计费日期 | 否 | 取材或清空时使用 |
| 备注 | 否 | 品系、周龄、特殊饲养要求等 |

### 6.5 IACUC 自动回填

当管理员已上传 IACUC 汇总表后，输入 IACUC 编号可自动匹配：

- 项目名称。
- 项目负责人。
- 实验负责人。
- 支撑经费。

如果自动匹配结果不符合实际，可以手动覆盖。

### 6.6 批量录入

适用场景：

- 同一项目一次入住多个笼位。
- 同一批次预约多个笼位。
- 同一批次取材。

操作步骤：

1. 点击 `多选录入`。
2. 选择多个笼位。
3. 填写统一的项目和日期信息。
4. 点击保存。

> [!IMPORTANT]
> 批量录入建议只选择同一 IACUC 或同一项目负责人下的笼位，避免误覆盖。

### 6.7 取材与最后计费日期

当动物已取材或结束饲养时：

1. 选择对应笼位。
2. 点击 `已取材`。
3. 填写最后计费日期。
4. 确认保存。

最后计费日期会影响当月计费明细，请按实际台账填写。

### 6.8 设为空

设为空用于将笼位恢复为空笼状态。

适用场景：

- 饲养结束。
- 预约取消。
- 信息录入错误后清理。

## 7. 饲养费核算

> 入口：`饲养费核算`

### 7.1 两种核算模式对比

| 模式 | 数据来源 | 适合场景 | 优点 | 注意事项 |
| --- | --- | --- | --- | --- |
| 动态笼位图（自动） | 笼位占用记录 | 日常系统内完整录入 | 自动、实时、追溯强 | 要求笼位录入及时准确 |
| 数量统计表（录入） | 纸质统计表 | 历史补录、纸质台账延续 | 贴合原有工作方式 | 依赖人工录入准确性 |

### 7.2 动态笼位图（自动）

操作步骤：

1. 打开 `饲养费核算`。
2. 选择 `动态笼位图（自动）`。
3. 选择结算月份。
4. 输入或选择项目负责人。
5. 查看汇总指标。
6. 导出 CSV 或 PDF。

汇总指标说明：

| 指标 | 说明 |
| --- | --- |
| 项目负责人 | 当前结算对象 |
| 伦理编号 | 该负责人名下相关 IACUC |
| 项目数量 | 参与汇总的 IACUC 数 |
| 免费笼数/天 | 根据负责人身份确定 |
| 累计笼日 | 当月总笼日 |
| 收费笼日 | 扣除免费额度后的笼日 |
| 应收金额 | 按阶梯规则计算后的金额 |

### 7.3 数量统计表（录入）

操作步骤：

1. 选择 `数量统计表（录入）`。
2. 点击 `新建`。
3. 填写结算月份、饲养间、房间号、管理员。
4. 填写 IACUC 编号。
5. 检查自动回填的项目负责人、实验负责人、项目名称、支撑经费。
6. 填写月初结余动物数和月初结余笼数。
7. 逐行录入每日新增、减少和结余。
8. 点击 `保存统计表`。
9. 查看结算预览并导出。

字段说明：

| 字段 | 说明 |
| --- | --- |
| 结算月份 | 统计表所属月份 |
| 饲养间 | 系统内房间，房间管理员只显示授权房间 |
| 房间号 | 可手动填写纸质表上的房间号 |
| IACUC 编号 | 计费和回填核心字段 |
| 月初结余动物数 | 月初动物数量 |
| 月初结余笼数 | 月初笼位数量，计费核心字段 |
| 变更行 | 每日新增、减少、结余和备注 |

### 7.4 计费规则

当前默认规则：

| 规则 | 数值 |
| --- | --- |
| PI 免费额度 | 20 笼/天 |
| 独立科研人员免费额度 | 10 笼/天 |
| 首单元阈值 | 前 160 笼 |
| 首单元价格 | 4.5 元/笼/天 |
| 超额单元价格 | 6.5 元/笼/天 |

计算口径：

```text
每日费用 = 扣除免费额度后的收费笼数 × 对应阶梯价格
月度费用 = 每日费用累加
```

### 7.5 导出文件

| 文件 | 用途 |
| --- | --- |
| 饲养明细 CSV | 后续复核、表格处理、归档 |
| 结算单 PDF | 对外确认或内部盖章流转 |

## 8. 数据管理与 IACUC 索引

> 入口：`数据管理`

### 8.1 IACUC 索引用途

IACUC 索引用于减少重复录入，提高项目字段一致性。

上传后可自动回填：

- 动物伦理编号。
- 动物实验名称。
- 项目负责人。
- 实验负责人。
- 支撑经费或项目来源。

### 8.2 CSV 文件要求

必须包含：

| 列名 | 用途 |
| --- | --- |
| 动物伦理编号 | IACUC 编号 |
| 动物实验名称 | 项目名称 |
| 项目负责人 | PI 或负责人 |
| 实验负责人 | 实验执行负责人 |

建议包含：

| 列名 | 用途 |
| --- | --- |
| 项目来源 | 支撑经费来源 |
| 支撑经费 | 结算单展示 |
| 经费来源 | 结算单展示 |

### 8.3 上传流程

1. 进入 `数据管理`。
2. 找到 `IACUC 索引`。
3. 选择 CSV 文件。
4. 点击上传。
5. 上传成功后查看索引条数。

### 8.4 数据处理规则

- 上传会重建后端实验申请索引表。
- 新录入时使用最新索引自动匹配。
- 已保存的历史占用记录不会被批量覆盖。

| 步骤 | 操作 | 目的 |

## 12. 常见问题与排查

### 12.1 房间管理员看不到某个房间

可能原因：

| 原因 | 处理 |
| --- | --- |
| 未授权该房间 | 管理员在账号管理中勾选房间 |
| 授权后未重新登录 | 退出并重新登录 |
| 房间已被删除 | 检查房间管理和操作日志 |

### 12.2 添加笼架后笼位数量不对

检查顺序：

1. 行数是否填写正确。
2. 列数是否填写正确。
3. 是否保存成功。
4. 是否在正确房间和笼架下查看。
5. 后端服务是否正常。

### 12.3 保存失败

常见原因：

| 原因 | 现象 | 处理 |
| --- | --- | --- |
| 权限不足 | 提示不能修改未授权房间 | 切换账号或调整授权 |
| 会话过期 | 操作失败或返回登录状态 | 重新登录 |
| 必填缺失 | 表单无法提交 | 补充必填字段 |
| 后端服务不可用 | 多个接口同时失败 | 检查服务或容器 |

### 12.4 导出结算文件失败

检查：

- 是否选择了结算月份。
- 项目负责人或 IACUC 是否有效。
- 当前账号是否有导出权限。
- 是否有可计费数据。
- 浏览器是否拦截下载。

### 12.5 群晖更新后版本没有变化

常见原因：

| 原因 | 处理 |
| --- | --- |
| Compose 固定了旧标签 | 修改 `CAGELEDGER_IMAGE_TAG` |
| 没有拉取新镜像 | 执行 `docker compose pull` |
| 只是停止后启动，没有重建 | 使用重新创建/构建流程 |
| 浏览器缓存 | 强制刷新页面 |

## 14. 部署、更新与发布

### 14.1 Docker Compose 部署

推荐目录：

```bash
mkdir -p /volume1/docker/cageledger
cd /volume1/docker/cageledger
```

启动：

```bash
docker compose pull
docker compose up -d
```

访问：

```text
http://群晖IP:5173
```

### 14.2 环境变量

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `CAGELEDGER_HOST` | `0.0.0.0` | 后端监听地址 |
| `CAGELEDGER_PORT` | `5173` | 后端监听端口 |
| `CAGELEDGER_DB` | `data/cageledger.sqlite` | SQLite 数据库路径 |
| `CAGELEDGER_IACUC_INDEX` | `data/iacuc-index.json` | 兼容索引文件路径 |
| `CAGELEDGER_ADMIN_USERNAME` | `admin` | 初始管理员账号 |
| `CAGELEDGER_ADMIN_PASSWORD` | `admin123` | 初始管理员密码 |
| `CAGELEDGER_REPOSITORY_URL` | `https://git.cellnucle.us/hugo/cageledger` | 项目仓库地址 |
| `CAGELEDGER_BRANCH` | `main` | 远端分支 |
| `CAGELEDGER_UPDATE_CHECK_ENABLED` | `false` | 是否启用远端更新检查 |
| `CAGELEDGER_GITEA_TOKEN` | 空 | 私有 Gitea 仓库更新检查使用的只读 token |

### 14.3 数据持久化

默认 Docker 命名卷：

```text
/app/data/cageledger.sqlite
```

建议改成宿主机目录挂载，便于备份：

```yaml
volumes:
  - ./data:/app/data
```

### 14.4 群晖图形界面更新建议

如果不方便 SSH，可在群晖 Container Manager 中操作：

1. 打开项目。
2. 停止当前容器。
3. 修改环境变量中的 `CAGELEDGER_IMAGE_TAG`。
4. 拉取新镜像。
5. 重新构建/重新创建项目。
6. 启动后进入系统管理页确认版本号。

> [!TIP]
> 如果 `image` 使用 `ghcr.io/hugo-yh/cageledger:${CAGELEDGER_IMAGE_TAG:-latest}`，但环境变量仍是旧版本，重新构建仍会使用旧镜像标签。

### 14.5 固定版本标签

建议生产环境使用固定版本：

```bash
CAGELEDGER_IMAGE_TAG=0.3.4 docker compose up -d
```

不建议长期依赖 `latest`，因为不利于回滚和追踪。

### 14.6 离线包部署

本机生成离线包：

```bash
npm run package:offline
```

生成文件示例：

```text
dist/CageLedger-offline-v0.3.4.tar.gz
```

NAS 离线构建：

```bash
tar -xzf CageLedger-offline-v0.3.4.tar.gz
cd CageLedger
docker compose -f docker-compose.offline.yml up -d --build
```

### 14.7 发布流程

标准发布流程：

```bash
npm run release:local -- --version 0.3.4 --push
```

标签推送后 GitHub Actions 会自动生成：

- GitHub Release。
- 离线源码包。
- GHCR 容器镜像。

补充说明：

- `SYSTEM_RELEASE_NOTES` 需要在发布前手动写入本次真实更新内容。
- `release:local` 会按“改版本 -> 校验 -> 打包 -> commit -> tag -> push”的固定顺序执行，避免标签打到错误提交。
- GHCR 镜像默认由 `v*` 标签触发，不再额外依赖 Release 事件二次触发。

### 16.2 权限建议

- 最小权限原则授权房间管理员。
- 人员离岗后及时禁用或删除账号。
- 管理员账号只分配给系统维护人员。
- 不共用个人账号。
