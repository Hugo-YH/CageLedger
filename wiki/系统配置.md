# 系统配置

本页整理部署参数、系统展示信息和更新检查相关配置。

## 核心环境变量

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `CAGELEDGER_HOST` | `0.0.0.0` | 服务监听地址 |
| `CAGELEDGER_PORT` | `5173` | 服务监听端口 |
| `CAGELEDGER_DB` | `data/cageledger.sqlite` | SQLite 路径 |
| `CAGELEDGER_IACUC_INDEX` | `data/iacuc-index.json` | 兼容索引文件路径 |
| `CAGELEDGER_ADMIN_USERNAME` | `admin` | 初始管理员账号 |
| `CAGELEDGER_ADMIN_PASSWORD` | `admin123` | 初始管理员密码 |

运行时代码默认密码是 `admin123`。`.env.example` 使用 `change-me` 作为部署示例值，部署时应明确替换。

## 页面展示信息

| 变量 | 用途 |
| --- | --- |
| `CAGELEDGER_ORGANIZATION` | 所属单位 |
| `CAGELEDGER_DEPARTMENT` | 所属科室 |
| `CAGELEDGER_DEVELOPER` | 开发人员 |
| `CAGELEDGER_CONTACT_EMAIL` | 联系邮箱 |
| `CAGELEDGER_LICENSE` | 开源协议 |
| `CAGELEDGER_COPYRIGHT` | 版权信息 |

## 更新检查配置

| 变量 | 说明 |
| --- | --- |
| `CAGELEDGER_REPOSITORY_URL` | 项目仓库地址 |
| `CAGELEDGER_BRANCH` | 默认分支，界面显示与元数据使用 |
| `CAGELEDGER_UPDATE_CHECK_ENABLED` | 是否开启更新检查 |
| `CAGELEDGER_GITEA_TOKEN` | 私有 Gitea 仓库 Release 检查只读 token |

更新检查以 Gitea 最新 Release 为准，`main` 上的新提交只有发布成 Release 后才会成为可安装更新。

## 本地缓存与刷新

关于系统页面提供“清理本地缓存并刷新”入口，适合处理这些情况：

- 发布后页面仍显示旧 UI
- 浏览器本地状态与后端数据不一致
- 静态模式演示后切回共享模式

## 安全建议

- 首次部署即更换默认管理员密码
- 令牌写入环境变量或 Secrets
- 外部访问通过内网或受控反向代理暴露
- 数据目录和环境变量文件纳入备份
