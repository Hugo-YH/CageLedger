# 系统配置

本页整理服务、数据库、系统信息和更新检查环境变量。

## 核心变量

| 变量                        | 默认值                   | 说明                                     |
| --------------------------- | ------------------------ | ---------------------------------------- |
| `CAGELEDGER_HOST`           | `0.0.0.0`                | Python 服务监听地址                      |
| `CAGELEDGER_PORT`           | `5173`                   | 生产服务端口；开发脚本将 API 改为 `5174` |
| `CAGELEDGER_DB`             | `data/cageledger.sqlite` | SQLite 路径                              |
| `CAGELEDGER_IACUC_INDEX`    | `data/iacuc-index.json`  | 兼容 IACUC 索引文件                      |
| `CAGELEDGER_ADMIN_USERNAME` | `admin`                  | 初始管理员用户名                         |
| `CAGELEDGER_ADMIN_PASSWORD` | `admin123`               | 初始管理员密码                           |
| `CAGELEDGER_APP_VERSION`    | `package.json` 版本      | 页面和 API 发布版本                      |
| `CAGELEDGER_REVISION`       | Git revision             | 构建提交号                               |
| `CAGELEDGER_DEV_ASSETS`     | 空                       | 开发 API 模式标记                        |

`.env.example` 使用部署示例值。正式部署设置强密码，并限制 `.env` 文件访问权限。

## Gitea 更新检查

| 变量                              | 说明                  |
| --------------------------------- | --------------------- |
| `CAGELEDGER_REPOSITORY_URL`       | Gitea 仓库 URL        |
| `CAGELEDGER_BRANCH`               | 构建分支元数据        |
| `CAGELEDGER_UPDATE_CHECK_ENABLED` | 开启最新 Release 检查 |
| `CAGELEDGER_GITEA_TOKEN`          | 私有仓库只读 token    |

系统更新以 Gitea 最新 Release 为准。`main` 的普通提交在创建 Release 后进入用户更新口径。

## 前端构建和缓存

- Vite 输出 `web-dist/` 哈希资源。
- Python 服务提供 ETag、gzip 和内存静态缓存。
- API 请求使用 `Cache-Control: no-store`。
- TanStack Query 默认 15 秒 staleTime。
- localStorage 只保存 `cageledger.ui.v2` 界面偏好。
- 导航栏“刷新”会清理本地 UI 缓存并重新加载页面。

页面版本和后端版本存在差异时，系统引导使用导航栏刷新按钮。

## 安全配置

- 首次部署更换默认管理员密码。
- Gitea token 使用只读权限。
- 外部访问启用 HTTPS 和访问控制。
- SQLite 数据目录、`.env` 和 Compose 文件纳入备份。
- 反向代理保留 Cookie、gzip 和必要的请求体大小。
