# 笼卡管理

笼卡管理负责把预约接收消息转成待接收批次，并驱动后续待进驻任务。

## 页面职责

- 识别预约接收文本
- 保存待接收批次
- 批量打印笼卡
- 确认实际接收
- 维护待接收批次状态
- 使用摄像头或短码识别笼卡

## 主流程

1. 粘贴预约接收消息
2. 检查批次号、IACUC、数量、房间和接收日期
3. 保存为待接收批次
4. 打印笼卡
5. 到货后点击“确认已接收”
6. 系统按已接收笼卡生成待进驻任务

每张实体笼卡在生成时分配持久唯一的 Animal Record ID。打印二维码、接收记录、待进驻任务、占用记录和公开查询沿用同一 ID。

## 状态流转

```text
未打印
→ 已打印
→ 已接收
```

已接收回退到已打印时，系统会同步撤回尚未真正入驻的开放待进驻任务。

## 识别重点

系统优先识别这些信息：

- 批次号
- IACUC 编号
- 购买单位
- 品系
- 数量
- 房间
- 接收日期

IACUC 匹配成功后，项目负责人和实验负责人会自动回填。

## 打印规则

- A4 每页 14 张
- 每张有效笼卡包含 Animal Record ID 二维码
- 打印数量不足 14 的倍数时可补空白卡，空白卡不分配有效 ID
- 供应商显示简称，原始值完整保存
- 打印建议先使用浏览器预览确认边距

## 笼卡识别

- 移动端通过“笼卡识别”打开摄像头
- 支持识别 4 位短码和兼容旧版完整链接
- 摄像头调用要求 HTTPS 或 localhost
- 查询结果展示动物、项目、房间和当前状态
- 公开查询不返回经费、报销、账号和审计数据
