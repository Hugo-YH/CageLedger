# 备份与维护

本页说明数据目录、备份频率和恢复关注点。

## 必备备份项

- SQLite 数据库目录
- `.env` 或部署环境变量
- `docker-compose.yml`
- 导出的结算文件

推荐直接备份整个部署目录和 `./data` 挂载目录。

## 建议频率

| 频率 | 内容 |
| --- | --- |
| 每日 | SQLite 数据库 |
| 每周 | 部署目录和配置 |
| 每次升级前 | 数据库、Compose、环境变量 |
| 每次结算后 | 导出的结算文件 |

## 群晖建议

- 使用 Hyper Backup
- 配合 Btrfs Snapshot
- 定期演练恢复

## 恢复关注点

- 数据库文件与 WAL 伴随文件保持同目录
- 环境变量与数据库一起恢复，避免管理员账号或更新检查配置漂移
- 恢复后先检查 `IACUC` 回填、登录和首页数据是否正常

## 日常维护建议

- 升级前先备份
- 定期检查磁盘空间
- 定期导出关键月份结算单
- 变更配置后记录当前版本和部署时间
