# 笼位与房间管理

## 业务对象

| 对象 | 说明 |
| --- | --- |
| 饲养间 | 实际房间或区域 |
| 笼架 | 饲养间中的架体 |
| 笼位 | 架体上的具体位置 |
| 占用记录 | 某笼位在一段时间内的项目占用 |

## 笼位状态

| 状态 | 说明 |
| --- | --- |
| 空笼 | 当前未占用 |
| 预约 | 已预约，尚未正式入住 |
| 在用 | 已入住并参与占用统计 |
| 已取材 | 项目结束，保留历史语义 |

## 待进驻动物

- 已打印笼卡在确认实际接收后，会按每张笼卡生成一条待进驻任务。
- 任务会提前出现在目标房间顶部，并按计划入驻日期分组。
- 提前排笼会把目标笼位占用为 `reserved`；正式入驻后转为 `active`，饲养费从实际入驻日开始。
- 系统管理员可在未预留前改签目标房间，系统保留改签记录。

## 维护原则

- “已取材”和“设为空”属于不同业务动作。
- 批量操作前先确认房间、笼架和目标笼位。
- 房间管理员只能操作授权房间。
- 涉及笼位显示异常时，先检查房间、笼架和笼位结构是否一致。
