# 笼位与房间管理

本页覆盖饲养间、笼架、笼位结构，以及待进驻和占用维护。

## 业务对象

| 对象 | 说明 |
| --- | --- |
| 饲养间 | 实际房间或区域 |
| 笼架 | 饲养间中的架体 |
| 笼位 | 架体上的具体位置 |
| 占用记录 | 某个笼位在一段时间内的占用历史 |

## 笼位状态

| 状态 | 含义 |
| --- | --- |
| 空 | 当前未占用 |
| 预约 | 已预约，尚未正式入驻 |
| 在用 | 已入驻并参与占用统计 |
| 已取材 | 项目结束，保留历史语义 |

“已取材”和“设为空”保留不同语义。

## 待进驻任务

- 已接收笼卡会按每张笼卡生成待进驻任务
- 任务会在目标房间顶部展示
- 预留后会占用目标笼位为 `reserved`
- 正式入驻后会转为 `active`
- 系统管理员可在任务未完成前改签目标房间

## 房间维护原则

- 一个房间可以设置默认动物和默认收费项目
- 房间管理员只维护授权房间
- 笼架编号支持维护实际业务编号，不要求连续
- 删除笼架或笼位前先确认没有活跃占用
