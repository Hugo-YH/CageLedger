# 项目结构

```text
.
├── index.html
├── server.py
├── src/
│   ├── app.js
│   └── styles.css
├── assets/
├── scripts/
├── wiki/
├── data/
├── .gitea/workflows/
├── Dockerfile
├── docker-compose.yml
└── docker-compose.offline.yml
```

## 关键文件

| 路径 | 说明 |
| --- | --- |
| `server.py` | API、SQLite、权限、迁移、审计 |
| `src/app.js` | 前端主逻辑 |
| `src/styles.css` | 前端样式 |
| `scripts/set_version.mjs` | 统一版本同步 |
| `scripts/release_local.sh` | 本地顺序化发布 |
| `wiki/` | 唯一正式文档源 |

## 目录原则

- `data/` 是运行数据，不作为常规代码修改对象。
- `dist/` 是打包产物，不手工编辑。
- `wiki/` 改动会自动同步到 Gitea Wiki。
