## 前端

| 路径                        | 责任                                                            |
| --------------------------- | --------------------------------------------------------------- |
| `src/main.tsx`              | React 根节点、QueryClient 和 UiProvider                         |
| `src/react/App.tsx`         | 会话门禁、公开扫码入口、工作区入口                              |
| `src/react/features/shell/` | 导航、权限可见性和业务页面懒加载                                |
| `src/react/features/`       | dashboard、intake、cages、billing、workflows、scanner、settings |
| `src/react/components/`     | 通用工作区和表格组件                                            |
| `src/react/api/`            | typed contracts、fetch client、Query hooks 和 query keys        |
| `src/contracts/`            | session、infrastructure、intake、quantity、billing 等响应类型   |
| `src/react/state/`          | 工作区 UI reducer 和界面偏好存储                                |
| `src/react/print/`          | 笼卡、二维码、数量统计表和结算单模板                            |
| `src/domain/`               | 可测试的纯业务函数                                              |
| `src/react/releaseNotes.ts` | 当前与历史更新记录的兼容聚合入口                                |
| `src/react/version.ts`      | 由版本脚本同步的前端版本                                        |
| `src/styles/`               | token、布局、组件、领域和响应式样式                             |
| `src/styles.css`            | 全局样式兼容导入入口                                            |

业务页面通过 `React.lazy` 拆包。服务端状态由 TanStack Query 管理，本地 reducer 只管理导航和显示偏好。

## 后端

| 路径                       | 责任                                          |
| -------------------------- | --------------------------------------------- |
| `server.py`                | 服务启动、应用装配和历史导出兼容入口          |
| `server_app/web/`          | Router、HTTP 生命周期、请求解析和响应         |
| `server_app/persistence/`  | schema 注册、迁移和索引                       |
| `server_app/domains/`      | 各领域 service、repository、rules 和 importer |
| `server_app/config.py`     | 环境变量、路径和系统元数据                    |
| `server_app/http.py`       | JSON、gzip、缓存头和计时                      |
| `server_app/static.py`     | `web-dist/`、ETag、gzip 和 SPA fallback       |
| `server_app/db.py`         | SQLite 连接辅助                               |
| `server_app/cache.py`      | 服务端短缓存和失效                            |
| `server_app/repositories/` | SQL、分页、结构化列和 payload 兼容            |
| `server_app/services/`     | 迁移期 service 兼容导出                       |
| `tests/`                   | Python 业务与兼容回归                         |

## 构建与运行目录

| 路径                                  | 类型                   | 规则                      |
| ------------------------------------- | ---------------------- | ------------------------- |
| `web-dist/`                           | Vite 生产构建          | 自动生成，Python 服务读取 |
| `dist/`                               | 离线发布包             | 自动生成                  |
| `data/`                               | SQLite、索引和运行数据 | 部署备份对象              |
| `node_modules/`                       | Node 依赖              | `npm ci` 生成             |
| `playwright-report/`、`test-results/` | 测试产物               | 自动生成                  |

这些目录不作为手工代码修改入口。

## 常用命令

```bash
npm run dev
npm run build
npm run check
npm run verify:full
npm run test:e2e
npm run smoke:api
npm run benchmark
npm run package:offline
npm run release:local -- --version X.Y.Z --push
```

发布检查、生产构建、浏览器回归、离线包和多架构镜像都由 Mac mini 本地脚本执行。Gitea 托管代码、Wiki、Release 离线包和容器镜像。

详细边界见 `docs/contracts/module-boundaries.md`。
