# 项目结构

本页描述当前仓库的实际结构和职责边界。

## 顶层结构

```text
.
├── index.html
├── server.py
├── server_app/
├── src/
├── web-dist/
├── assets/
├── scripts/
├── wiki/
├── docs/
├── data/
├── .gitea/workflows/
├── Dockerfile
├── docker-compose.yml
└── docker-compose.offline.yml
```

## 关键目录职责

| 路径 | 职责 |
| --- | --- |
| `server.py` | HTTP 入口、路由分发、兼容主入口 |
| `server_app/config.py` | 环境变量、路径、版本和更新检查配置 |
| `server_app/db.py` | SQLite 连接与初始化辅助 |
| `server_app/cache.py` | 服务端短时缓存与相关辅助 |
| `server_app/repositories/` | 表级 SQLite 读写、分页、payload dump/load |
| `server_app/services/` | 接收、待进驻、数量统计表、结算流程等领域服务 |
| `src/main.tsx` | React 应用入口和 Provider 装配 |
| `src/react/` | React 应用壳、typed API、UI state 和业务视图 |
| `src/react/api/` | typed API、TanStack Query hooks 和查询键 |
| `src/react/state/` | UI reducer 和界面偏好存储边界 |
| `src/react/features/` | 按业务域懒加载的 React 页面与功能区 |
| `src/domain/` | 笼位、笼卡和数量统计表纯业务函数 |
| `src/react/releaseNotes.ts` | 系统更新记录与发布校验数据源 |
| `src/styles.css` | 全局样式 |
| `web-dist/` | Vite 哈希构建产物，不手工编辑 |
| `scripts/` | 版本同步、发布、离线打包、API 冒烟等脚本 |
| `wiki/` | 正式公开文档源 |
| `docs/` | 分析、契约、迁移、进度文档 |
| `data/` | 运行数据目录 |

## 目录原则

- `data/` 只存运行数据，不作为常规代码修改对象
- `dist/` 是打包产物目录，不手工编辑
- `web-dist/` 是前端构建产物目录，不手工编辑
- `wiki/` 变更会同步到 Gitea Wiki
- `docs/` 用于工程化推进，不替代 `wiki/`
- 正式前端只从 `src/main.tsx` 进入 React 应用，业务数据统一通过 Python API 读写

## 常用入口

| 用途 | 命令或路径 |
| --- | --- |
| 开发模式 | `npm run dev` |
| 生产构建 | `npm run build` |
| 语法检查 | `npm run check` |
| 浏览器回归 | `npm run test:e2e` |
| 性能基准 | `npm run benchmark` |
| API 冒烟 | `npm run smoke:api` |
| 离线打包 | `npm run package:offline` |
| 本地发布 | `npm run release:local -- --version X.Y.Z --push` |
