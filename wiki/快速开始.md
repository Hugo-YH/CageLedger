# 快速开始

本页提供首次开发运行和首次业务配置的最短路径。

## 开发运行

环境要求：Node.js `>=20.19.0`、Python 3。

```bash
git clone https://git.cellnucle.us/hugo/cageledger.git
cd cageledger
npm ci
npm run dev
```

开发模式同时启动：

| 服务       | 地址                    | 作用                  |
| ---------- | ----------------------- | --------------------- |
| Vite       | `http://localhost:5173` | React 页面与热更新    |
| Python API | `http://127.0.0.1:5174` | SQLite、鉴权和 `/api` |

Vite 自动代理 `/api`。默认数据库是 `data/cageledger.sqlite`。

## 生产运行

```bash
npm start
```

该命令构建 `web-dist/`，随后由 Python 服务在 `http://localhost:5173` 提供页面和 API。

## 默认账号

```text
用户名：admin
密码：admin123
```

正式部署通过 `CAGELEDGER_ADMIN_USERNAME` 和 `CAGELEDGER_ADMIN_PASSWORD` 设置初始管理员，并在首次登录后使用强密码。

## 首次业务配置

| 顺序 | 操作                               | 验收结果                                           |
| ---- | ---------------------------------- | -------------------------------------------------- |
| 1    | 登录管理员账号                     | 当前用户显示系统管理员                             |
| 2    | 数据管理上传 IACUC 汇总 CSV        | 伦理号可回填项目、PI、实验负责人、项目来源和有效期 |
| 3    | 房间管理建立设施、房间、笼架和笼位 | 笼位管理可以选择房间和笼架                         |
| 4    | 账号管理创建房间管理员并授权       | 房间管理员只维护授权设施数据                       |
| 5    | 数据管理配置 PI 身份与减免额度     | 结算可以执行 PI 减免分配                           |
| 6    | 建立一条接收或数量统计表样本       | 主业务数据可以保存和刷新读取                       |
| 7    | 生成结算并发起流程                 | 流程中心出现同月同 PI 台账                         |

## 推荐验收路径

### 笼卡到入驻

1. 笼卡管理识别预约消息。
2. 保存待接收批次并打印笼卡。
3. 确认已接收。
4. 笼位管理处理待进驻任务。
5. 预留或正式入驻。
6. 扫描笼卡二维码确认 Animal Record ID 可查询。

### 月度结算到报销

1. 饲养费管理选择动态笼位图或数量统计表入口。
2. 按月份和项目负责人生成结算预览。
3. 导出结算单并发起流程。
4. 流程中心登记报销并推进状态。
