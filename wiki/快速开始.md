# 快速开始

本页面向首次试用系统的管理员，目标是在最短路径内完成可用闭环。

## 运行模式

| 模式 | 命令 | 适用场景 |
| --- | --- | --- |
| 开发模式 | `npm run dev` | 同时启动 Vite 与 Python API，数据写入 SQLite |
| 生产构建 | `npm run build` | 生成由 Python 服务提供的 `web-dist` 哈希资源 |

默认访问地址：

```text
http://localhost:5173
```

默认管理员：

```text
用户名：admin
密码：admin123
```

## 首次配置顺序

| 顺序 | 操作 | 结果 |
| --- | --- | --- |
| 1 | 登录管理员账号并修改密码 | 固定系统管理员身份 |
| 2 | 在“数据管理”上传 IACUC 汇总表 CSV | 录入 IACUC 后可自动回填项目、负责人和经费来源 |
| 3 | 在“房间管理”建立饲养间、笼架和笼位 | 形成实际设施结构 |
| 4 | 在“账号管理”创建房间管理员并授权 | 建立权限边界 |
| 5 | 用“笼卡管理”或“笼位管理 / 饲养费管理”录入测试数据 | 建立接收、占用或数量统计表样本 |
| 6 | 生成测试结算单并推进一次流程 | 验证结算、导出和流程中心链路 |

## 推荐试用路径

### 路径 A：接收与入驻

1. 打开 [[笼卡管理]]。
2. 粘贴预约接收消息并保存待接收批次。
3. 打印笼卡并点击“确认已接收”。
4. 打开 [[笼位与房间管理]] 查看待进驻任务。
5. 预留或正式入驻到目标笼位。

### 路径 B：月度结算

1. 打开 [[饲养费核算]]。
2. 选择“动态笼位图”或“数量统计表”入口。
3. 生成结算预览，导出 PDF / CSV。
4. 发起流程并在流程中心推进。

## 试用前检查

```bash
npm run check
```

如果页面行为异常，先确认 `5173` 端口上的进程就是当前仓库启动的服务。
