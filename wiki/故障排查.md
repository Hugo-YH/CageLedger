# 故障排查

本页按真实故障现象组织，优先给出定位顺序。

## 页面打不开或空白

1. 检查 `5173` 端口上的进程是否是当前仓库启动的服务。
2. 检查容器或本地服务是否仍在运行。
3. 刷新浏览器，必要时在“关于系统”执行“清理本地缓存并刷新”。

本地检查：

```bash
lsof -i :5173
```

Docker 检查：

```bash
docker compose ps
docker logs cageledger
```

## 登录失败

检查顺序：

1. 默认密码是否已修改
2. 环境变量中的管理员账号密码是否生效
3. 数据库是否被替换或重建

## IACUC 无法回填

1. 检查 CSV 是否包含 `动物伦理编号`、`动物实验名称`、`项目负责人`、`实验负责人`
2. 检查 IACUC 编号是否与 CSV 中完全一致
3. 检查上传后是否刷新页面
4. 检查当前账号是否能访问数据管理结果

## 结算金额异常

按以下顺序排查：

1. IACUC 是否正确
2. 项目负责人和负责人身份是否正确
3. 动态笼位图和数量统计表入口是否混用
4. 入驻日期、结束日期、转入转出是否完整
5. 同月流程是否基于正确版本重新生成

## 系统更新没有新版本

检查顺序：

1. 是否已经创建 Gitea Release
2. `CAGELEDGER_UPDATE_CHECK_ENABLED` 是否为 `true`
3. 私有仓库是否配置了 `CAGELEDGER_GITEA_TOKEN`
4. 当前版本号是否已经等于最新 Release

## Wiki 未同步

检查顺序：

1. 是否改动了 `wiki/**`
2. 是否已经推送到 `main`
3. `sync-wiki.yml` 是否成功执行
4. Gitea token 是否具备写入 Wiki 的权限

## SQLite WAL 文件没看到

`cageledger.sqlite-wal` 和 `cageledger.sqlite-shm` 是 SQLite WAL 模式的运行时伴随文件。数据库关闭、checkpoint 完成或文件浏览器未显示隐藏目录时，这两个文件都可能暂时看不到。
