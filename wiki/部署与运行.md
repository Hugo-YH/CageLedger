# 部署与运行

本页覆盖本地、Docker、群晖 NAS 和升级维护的标准路径。

## 本地运行

```bash
npm run dev
```

该命令在 `5173` 启动 Vite，在 `5174` 启动 Python API，并由 Vite 代理 `/api`。生产构建使用：

```bash
npm run build
```

构建产物写入 `web-dist/`，由 Python 服务直接提供。正式运行和离线包不要求目标机器安装 Node.js。

默认访问：

```text
http://localhost:5173
```

SQLite 默认路径：

```text
data/cageledger.sqlite
```

## Docker Compose 在线部署

### 1. 准备目录

```bash
mkdir -p /volume1/docker/cageledger
cd /volume1/docker/cageledger
git clone https://git.cellnucle.us/hugo/cageledger.git .
```

### 2. 启动服务

```bash
docker compose pull
docker compose up -d
```

### 3. 访问系统

```text
http://服务器IP:5173
```

## 镜像与 Compose

| 文件 | 用途 |
| --- | --- |
| `docker-compose.yml` | 在线部署，优先使用私有 Gitea 镜像 |
| `docker-compose.offline.yml` | 离线或源码构建场景 |

Dockerfile 使用 Node 构建前端、Python 运行服务的多阶段镜像。运行阶段只包含 `web-dist` 和 Python 服务依赖。

正式镜像地址：

```text
git.cellnucle.us/hugo/cageledger:<tag>
```

固定版本示例：

```bash
CAGELEDGER_IMAGE_TAG=0.5.25 docker compose up -d
```

## 离线包直接运行

离线包已经包含 `web-dist`，目标机器只需 Python 3，无需安装 Node.js：

```bash
tar -xzf CageLedger-offline-v0.5.25.tar.gz
cd CageLedger
python3 server.py
```

系统默认监听 `5173`。历史报销 Excel 导入依赖 `openpyxl`，需要该功能时安装：

```bash
python3 -m pip install openpyxl
```

离线包同时保留 Docker 源码构建入口；使用 `docker-compose.offline.yml` 时，构建环境需要预先具备对应 Node 与 Python 基础镜像。

## 数据持久化

容器内数据目录：

```text
/app/data
```

当前仓库默认 Compose 使用 Docker named volume：

```yaml
volumes:
  - cageledger-data:/app/data
```

如果需要把数据放到宿主机可见目录，可改成 bind mount：

```yaml
volumes:
  - ./data:/app/data
```

关注文件：

- `cageledger.sqlite`
- `iacuc-index.json`
- SQLite WAL 运行时文件：`cageledger.sqlite-wal`、`cageledger.sqlite-shm`

## 环境变量

部署时建议至少配置：

| 变量 | 说明 |
| --- | --- |
| `CAGELEDGER_ADMIN_USERNAME` | 初始管理员账号 |
| `CAGELEDGER_ADMIN_PASSWORD` | 初始管理员密码 |
| `CAGELEDGER_UPDATE_CHECK_ENABLED` | 是否启用更新检查 |
| `CAGELEDGER_GITEA_TOKEN` | 私有 Gitea 仓库 Release 检查只读 token |

完整清单见 [[系统配置]]。

`.env.example` 中的管理员密码示例是 `change-me`，部署时应替换成真实强密码。

## 升级方式

### 代码仓库部署

```bash
git pull --ff-only origin main
docker compose pull
docker compose up -d
```

### 指定版本镜像升级

```bash
CAGELEDGER_IMAGE_TAG=0.5.25 docker compose pull
CAGELEDGER_IMAGE_TAG=0.5.25 docker compose up -d
```

升级前建议备份数据库和环境变量。

## 版本回滚

回滚容器版本时固定上一版镜像 tag，并继续挂载原数据卷：

```bash
CAGELEDGER_IMAGE_TAG=0.5.25 docker compose pull
CAGELEDGER_IMAGE_TAG=0.5.25 docker compose up -d
```

当前 SQLite 迁移只增加字段和索引，v0.5.25 可以忽略新增结构并读取原业务 payload。回滚前继续备份数据库；未来出现删除字段或重写数据的迁移时，应同时恢复升级前数据库备份。

回滚后检查：

1. `/api/health` 返回当前旧版本。
2. 管理员和房间管理员均可登录。
3. 笼卡、笼位、数量统计表和流程中心可读取历史数据。
4. 指定月份的结算金额与升级前导出结果一致。

## 群晖建议

- 部署目录固定在 `/volume1/docker/cageledger`
- 数据目录与 Compose 文件放在同一层便于备份
- 使用 Hyper Backup 或 Btrfs Snapshot 管理数据目录
- 升级前导出当前 `docker-compose.yml`、`.env` 和数据库
