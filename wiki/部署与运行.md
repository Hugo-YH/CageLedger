# 部署与运行

本页覆盖开发、生产、Docker、群晖 NAS、离线包和升级回滚。

## 运行模型

| 模式     | 前端                                     | API                                | 数据库                        |
| -------- | ---------------------------------------- | ---------------------------------- | ----------------------------- |
| 开发     | Vite `5173`                              | Python `5174`，由 Vite 代理 `/api` | `data/cageledger.sqlite`      |
| 本地生产 | Python `5173` 提供 `web-dist/` 和 `/api` | 同一进程                           | `data/cageledger.sqlite`      |
| Docker   | 容器 `5173`                              | 同一容器                           | `/app/data/cageledger.sqlite` |

## 开发模式

```bash
npm ci
npm run dev
```

- 页面：`http://localhost:5173`
- API：`http://127.0.0.1:5174`
- 临时隔离数据库：`CAGELEDGER_EPHEMERAL_DB=1 npm run dev`
- 自定义 API：`CAGELEDGER_API_ORIGIN=http://127.0.0.1:5174 npm run dev`

## 本地生产模式

```bash
npm start
```

`npm start` 先执行 Vite 构建，再启动 Python 服务。构建输出是 `web-dist/`，生产服务在 `5173` 同时提供 SPA、静态资源和 API。

也可以分步执行：

```bash
npm run build
python3 server.py
```

## Docker 镜像

Dockerfile 使用多阶段构建：

1. Node 22 阶段执行 `npm ci` 和 `npm run build`。
2. Python 3.13 阶段安装 `openpyxl`，复制后端和 `web-dist/`。
3. 运行阶段只启动 `python3 server.py`。

正式镜像：

```text
git.cellnucle.us/hugo/cageledger:<tag>
```

## Docker Compose

```bash
mkdir -p /volume1/docker/cageledger
cd /volume1/docker/cageledger
git clone https://git.cellnucle.us/hugo/cageledger.git .
cp .env.example .env
docker compose pull
docker compose up -d
```

访问：`http://NAS地址:5173`。

固定版本：

```bash
CAGELEDGER_IMAGE_TAG=<version> docker compose pull
CAGELEDGER_IMAGE_TAG=<version> docker compose up -d
```

`docker-compose.yml` 使用 Gitea 镜像。`docker-compose.offline.yml` 用于离线源码构建。

## 数据持久化

容器内数据目录：

```text
/app/data
```

默认 Compose 使用 named volume：

```yaml
volumes:
  - cageledger-data:/app/data
```

群晖文件管理器需要直接查看数据时，使用 bind mount：

```yaml
volumes:
  - /volume1/docker/cageledger/data:/app/data
```

关注文件：

- `cageledger.sqlite`
- `iacuc-index.json`
- `cageledger.sqlite-wal`
- `cageledger.sqlite-shm`

WAL 和 SHM 是运行时伴随文件，checkpoint 或服务停止后可能消失。备份以 SQLite 一致性为准，详见 [[备份与维护]]。

## 离线包

```bash
npm run package:offline
```

输出：

```text
dist/CageLedger-offline-v<version>.tar.gz
```

离线包包含 `web-dist/`。目标机器直接运行：

```bash
tar -xzf CageLedger-offline-v<version>.tar.gz
cd cageledger
python3 -m pip install openpyxl
python3 server.py
```

运行目标机器无需 Node.js。Excel 历史报销导入依赖 `openpyxl`。

## 升级

升级前备份 SQLite、`.env` 和 Compose 文件。

```bash
git pull --ff-only origin main
CAGELEDGER_IMAGE_TAG=<new-version> docker compose pull
CAGELEDGER_IMAGE_TAG=<new-version> docker compose up -d
```

服务启动时自动执行幂等 SQLite schema 初始化、字段补齐、索引创建和必要数据回填。旧数据库无需人工 SQL 操作。

升级后检查：

```bash
curl http://127.0.0.1:5173/api/health
docker logs --tail 200 cageledger
```

再验证管理员登录、房间管理员权限、笼卡、笼位、统计表、结算和流程中心。

## 回滚

1. 停止写入并备份当前数据库。
2. 固定上一版镜像 tag。
3. 启动上一版容器并挂载原数据目录。
4. 检查 `/api/health` 和关键业务链。

新增字段和索引通常支持旧版本忽略。涉及删除字段、重写数据或不可逆迁移的版本需要同时恢复升级前数据库备份，发布说明应明确标注。

## 群晖建议

- 项目目录固定为 `/volume1/docker/cageledger`。
- 数据使用 `/volume1/docker/cageledger/data` bind mount，便于备份和迁移。
- 使用 Hyper Backup 或 Btrfs Snapshot 管理数据目录。
- 外部访问通过受控反向代理和 HTTPS。
- 升级使用明确版本 tag，避免长期跟随 `latest`。
