# 开发规范

本页约束 React 迁移后的代码修改、测试、文档和发布。

## 开发基线

- Node.js `>=22.12.0`，Python 3.13。
- 依赖安装使用 `npm ci` 和 `python3 -m pip install -r requirements-dev.txt`。
- React 页面使用 TypeScript。
- 服务端业务数据使用 TanStack Query。
- 本地 reducer 管理界面偏好。
- Python 后端保持 HTTP、service、repository 分层。
- SQLite 迁移保持幂等和旧库兼容。

## 前端规则

1. 页面通过 typed API hooks 访问服务端。
2. 查询键集中在 `queryKeys.ts`，mutation 精确失效相关根键。
3. 高频输入保留局部草稿，输入过程中避免整页查询失效。
4. 跨页面可复用计算进入 `src/domain/` 并补 Vitest。
5. 业务页面通过现有懒加载入口接入。
6. 预览与打印共用 `src/react/print/` 模板。
7. 站内通知和确认弹层承载反馈。
8. 颜色和按钮层级遵守 `docs/contracts/ui-color-system.md`。

## 后端规则

1. handler 负责鉴权、参数、状态码和响应。
2. service 负责业务校验、事务、审计和缓存失效。
3. repository 负责 SQL、分页、索引和 payload 兼容。
4. 高频列表使用结构化列和服务端分页。
5. 写入接口返回最新对象和受影响对象。
6. schema 新增字段和索引时补自动迁移与旧数据回填。

## 业务规则

- IACUC 是笼卡、占用、统计表和结算的核心键。
- Animal Record ID 是笼卡实例的持久唯一标识。
- `项目来源` 是 `funding` 的 source of truth。
- 动态笼位图和数量统计表是并存结算入口。
- PI 减免需要同时考虑总额度、IACUC 有效期、优先减免和逐日分配。
- 结算流程记录单据状态，报销台账记录资金回收状态。
- 权限变化同时验证前端入口和后端校验。

## 工具链

- Prettier：TS、TSX、CSS、JSON、YAML 和活动 Markdown。
- ESLint：TypeScript、React Hooks、React Refresh 和 JSX 可访问性。
- Stylelint：CSS 标准语法、未知值和重复声明。
- Markdownlint：标题、列表、代码块和链接。
- Ruff：Python 格式、import 和 lint。
- Playwright + axe：关键业务链和 serious/critical 可访问性回归。

基础质量检查为 `npm run check`，完整发布验证为 `npm run verify:full`。详细规则见 `CONTRIBUTING.md`、`docs/contracts/code-quality.md` 和 `docs/contracts/testing-strategy.md`。

## 产物与数据

- `web-dist/`、`dist/`、`node_modules/` 由工具生成。
- `data/` 保存 SQLite 和运行数据。
- 性能测试使用临时目录。
- 运行库和构建产物不进入常规代码提交。

## 文档同步

- 用户操作、部署、配置和权限更新 `wiki/`。
- 模块、状态、API 和颜色规范更新 `docs/contracts/`。
- `README.md` 保持轻量入口。
- 已完成迁移资料保留在 `docs/archives/`。

## 发布

1. 更新 `src/react/releaseNotes.ts`，版本号旁写发布日期和时间。
2. 运行 `npm run release:local -- --version X.Y.Z --push`。
3. 确认 commit、`vX.Y.Z` tag、Gitea Release 和容器镜像一致。
4. 确认本地发布脚本已同步 `wiki/` 到 Gitea Wiki。

完整代理规则见仓库根目录 `AGENTS.md`。
