# 开发规范

## 基本原则

1. 数据正确优先。
2. 权限边界清晰。
3. 前后端链路一致。
4. 优先沿用现有写法。
5. 少引入新框架和新工具链。

## 常用检查

```bash
npm run check
```

涉及交互、缓存、权限、导出时，再实际打开页面验证。

## 代码约束

- 不随意重写 `src/app.js` 整体结构。
- 不删除历史兼容逻辑。
- 不直接修改 `data/` 和 `dist/`。
- 前端通知走站内通知和站内确认弹层。
- 修改接口、部署、发布、业务规则时同步更新 `wiki/`。

## 发布约束

- 只用仓库脚本改版本。
- 不复用旧 tag 修补新内容。
- Gitea 最新 Release 才代表可安装更新。
